## 题目描述

给定一个只包括 `'('，')'，'{'，'}'，'['，']'` 的字符串 `s` ，判断字符串是否有效。

有效字符串需满足：

1. 左括号必须用相同类型的右括号闭合。
2. 左括号必须以正确的顺序闭合。

**示例 1：**

> 输入：s = "()"
> 
> 输出：true

**示例 2：**

> 输入：s = "()[]{}"
> 
> 输出：true

**示例 3：**

> 输入：s = "(]"
> 
> 输出：false

**示例 4：**

> 输入：s = "([)]"
> 
> 输出：false

**示例 5：**

> 输入：s = "{[]}"
> 
> 输出：true


## 题解

### 算法思路

和**表达式求值**类似地，括号的匹配也是栈来进行实现。
- 对于每一个左括号，可以直接入栈
- 对于每一个右括号，需要检查栈顶元素是否为对应的左括号。如果是，则将该左括号出栈；如果不是，则返回 `false`

最后栈必须为空（即所有左括号都被匹配）才能符合括号匹配的要求

一些官方题解中的优化：
- 使用 `Map` 来存储 `<右括号, 左括号>` 的键值对（虽然可以简化代码但是感觉没必要）
- 括号若匹配，则原字符串的长度一定是**偶数**，如果开始判断长度为奇数，则可以直接返回 `false`

#### 错误的思路

使用计数器记录左括号的数量：
- 遇到左括号则计数器自增
- 遇到右括号则计数器自减，如果计数器变成小于0，则说明有孤立的右括号无法匹配，返回 `false`
- 遍历完字符串之后，需要所有的计数器都为0

这样的思路，遇到示例四 `"([)]"` 的情况就会判断错误。刚开始以为，错是错在没有考虑括号之间的优先级顺序。但事实上错的地方是，由于括号之内一定是一个**完整的表达式**，也就是递归定义，**括号内部一定是一个完整的括号匹配序列**。因此示例四这种左右括号里面括号不匹配的情况就是不符合题意的。（和 `"(())"` 或者 `"[[]]"` 这样的情况不同。）

### 代码实现

```java
class Solution {
    public boolean isValid(String s) {
        int n = s.length();
        if (n % 2 == 1) {
            return false;
        }

        Map<Character, Character> pairs = new HashMap<Character, Character>() {{
            put(')', '(');
            put(']', '[');
            put('}', '{');
        }};
        Deque<Character> stack = new LinkedList<Character>();
        for (int i = 0; i < n; i++) {
            char ch = s.charAt(i);
            if (pairs.containsKey(ch)) {
                if (stack.isEmpty() || stack.peek() != pairs.get(ch)) {
                    return false;
                }
                stack.pop();
            } else {
                stack.push(ch);
            }
        }
        return stack.isEmpty();
    }
}
```
