红米ac2100使用OpenWrt获取教育网ipv6地址

2021-11-15 877 字
路由器 折腾

OpenWrt 安装

可喜可贺，OpenWrt官方出了Redmi ac2100的官方维护包，并且如果不再继续折腾的话，不用breed直接刷入也可以。但是相应地，比起一些第三方的魔改OpenWrt就少了一些内核级别的功能（比如Turbo ACC和CPU超频）。

第三方OpenWrt之前用过寒枫叶做的固件。CPU超频到了1000MHz，也编译进去了Turbo ACC，但是可惜他的自建网盘无法访问，恩山论坛帖子里面的蓝奏盘链接也进不去。于是还是决定用官方的固件养老吧。

官方固件下载

OpenWrt Firmware Selector这个站十分人性化，可以直接输入AC2100搜索获得最新的OpenWrt固件。下载其中的xxx-kernel1.bin、xxx-rootfs0.bin、xxx-sysupgrade.bin。

破解SSH

一些破解SSH的方法挺多的，这里就直接贴：

原厂固件的WEB管理界面，地址格式如下：

http://192.168.31.1/cgi-bin/luci/;stok=<STOK>/web/home#router

复制其中的STOK部分，在以下的地址中替换并访问：

http://192.168.31.1/cgi-bin/luci/;stok=<STOK>/api/misystem/set_config_iotdev?bssid=Xiaomi&user_id=longdike&ssid=-h%3B%20nvram%20set%20ssh_en%3D1%3B%20nvram%20commit%3B%20sed%20-i%20's%2Fchannel%3D.*%2Fchannel%3D%5C%22debug%5C%22%2Fg'%20%2Fetc%2Finit.d%2Fdropbear%3B%20%2Fetc%2Finit.d%2Fdropbear%20start%3B

返回{"code":0}即代表成功。破解SSH之后root的密码并不是确定的。这里可以使用类似于上面的方法，将root的密码重置为admin。

http://192.168.31.1/cgi-bin/luci/;stok=<STOK>/api/misystem/set_config_iotdev?bssid=Xiaomi&user_id=longdike&ssid=-h%3B%20echo%20-e%20'admin%5Cnadmin'%20%7C%20passwd%20root%3B

固件刷入

将三个固件文件传入/tmp目录，之后随便用一个SSH客户端，在/tmp中执行以下命令：

nvram set uart_en=1&&nvram set bootdelay=5&&nvram set flag_try_sys1_failed=1&&nvram commit
mtd write /tmp/xxx-kernel1.bin kernel1
mtd -r write /tmp/xxx-rootfs0.bin rootfs0

重启之后，再准备刷入xxx-sysupgrade.bin，在固件目录执行以下命令：

sysupgrade -v xxx-sysupgrade.bin

ipv6 相关

环境说明

所在学校是南京大学，学校的校园网ipv4需要登陆进行认证获得，而ipv6则不需要任何认证，宿舍的网口插入之后，Windows就可以直接获得原生的教育网ipv6地址。

实现原理

以上环境最适合实现的就是穿透模式。也就是路由器本身没有ipv6地址，而所有连接路由器（无论有线无线）的设备，可以像直接插入网口一样，直接获得ipv6地址。

这个模式可能唯一的缺点就是，无法通过ipv6访问路由器本身的管理页面~~（所以呢？）~~

实现过程

OpenWrt中所有关键词带有ebtables的插件（按说ipv4相关的似乎并不需要）。

在System-startup的本地脚本中，加入以下命令：

sleep 10
/sbin/insmod ebtables
/sbin/insmod ebtable_broute
/sbin/insmod /lib/modules/【5.4.154】/ebtable_broute.ko
ebtables -t broute -A BROUTING -p ! ipv6 -j DROP -i 【wan】
brctl addif 【br-lan】 【wan】

其中使用【】括起来的，可能会根据设备的不同需要替换（内核版本、wan口接口名称、lan口接口名称）。其中内核版本可以使用uname -r命令查看，WEB管理页面应该也有。而两个端口则可以通过ifconfig命令获取（看与其绑定的地址应该可以看得出来）

如此，重启之后（路由器以及设备网卡）应该就可以获得ipv6地址了！
