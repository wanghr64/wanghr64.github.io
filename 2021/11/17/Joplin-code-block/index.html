Joplin代码块美化

2021-11-17 401 字
折腾 Joplin CSS

前言

Joplin作为一个开源的、使用Markdown的、跨平台的、支持WebDAV同步的、支持网页剪辑的笔记软件，可以说满足了我的大多数应用场景（没有脑图，但是问题不大）。但是开源软件总是会有这个那个的小毛病，比如说代码块的效果就十分差劲：

  • 无论是在编辑器中，还是在渲染过的Markdown中，都没有使用等宽字体，而是使用了中文Windows中经常作为缺省字体的某非等宽衬线字体
  • 在渲染过的Markdown中，代码块不支持自动换行。这一点在Android客户端中是正常的，因此就是Windows客户端的锅

解决方案

等宽字体

编辑器

编辑器修改为等宽字体，可以非常直观地在编辑器等宽字体族直接填入想要的字体（中间有空格也不需要使用单引号）

Markdown渲染

在显示高级选项中，可以修改适用于已渲染 Markdown 的自定义样式表。在打开的userstyle.css中，添加如下代码：

code {
	font-family: "Hack";
}

也就是将字体族换为自己想要的字体。

自动换行

编辑器中的代码块，是支持自动换行的。但是在渲染的Markdown中，就会出现这样的情况：

解决方法同样是修改之前的userstyle.css，在之前的code中加入如下一行white-space: pre-wrap;，最终效果如下：

code {
	font-family: "Hack";
	white-space: pre-wrap;
}
